😫 遇到 ".NET SDK 未检测到" 的问题？

✅ 最简单的解决方法：

   👉 直接双击运行这个文件:
   
      ⚡一键解决并编译.bat

   这个脚本会：
   ✓ 自动找到 .NET SDK 安装位置
   ✓ 自动编译生成 exe 程序
   ✓ 询问是否立即启动
   
   无需重启电脑！无需手动设置！

═══════════════════════════════════════════════════════════

📖 详细说明:

   问题原因:
   .NET SDK 已安装，但系统环境变量还没生效，
   导致命令行找不到 dotnet 命令。

   常规解决方法:
   - 重启电脑 (让环境变量生效)
   
   快速解决方法:
   - 用我们的脚本直接找到 .NET 的完整路径来编译
   - 不依赖环境变量，立即可用！

═══════════════════════════════════════════════════════════

🚀 操作步骤:

   1. 双击: ⚡一键解决并编译.bat
   
   2. 等待 1-2 分钟编译完成
   
   3. 选择 "Y" 立即启动程序
   
   4. 在系统托盘配置 Flomo API
   
   5. 按 Ctrl+Alt+F 开始使用！

═══════════════════════════════════════════════════════════

❓ 如果一键脚本也不行?

   说明 .NET SDK 可能没有正确安装，请:
   
   1. 访问: https://dotnet.microsoft.com/zh-cn/download/dotnet/6.0
   
   2. 下载 ".NET 6.0 SDK" (Windows x64 版本)
   
   3. 安装完成后，再次运行 "⚡一键解决并编译.bat"

═══════════════════════════════════════════════════════════

💡 提示: 
   如果想让 dotnet 命令全局可用（可选），
   安装完成后重启电脑即可。
   但对于编译这个项目，不重启也完全可以！
